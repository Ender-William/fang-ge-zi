## 1. 产品概述

### 1.1 产品定义

| 项目 | 内容 |
|------|------|
| **产品名称** | 鸽巢管家 |
| **产品类型** | 移动应用（Android原生应用） |
| **定位** | 面向养鸽爱好者的鸽子信息管理与家族谱系追踪工具 |
| **核心愿景** | 让每一位养鸽人都能轻松记录和管理鸽子的位置与血统，不再为记不住"鸽子在哪、是谁的后代"而烦恼 |
| **品牌调性** | 朴实、可靠、温暖、简单 |

### 1.2 产品愿景

鸽巢管家致力于成为中国养鸽人群首选的鸽子信息管理工具。对于以中老年为主的养鸽群体，我们希望通过简单易用的数字化工具，替代传统的纸质笔记本和凭记忆管理的方式，帮助用户：

- 随时随地记录每只鸽子所在的棚舍位置
- 清晰追溯每只鸽子的血统来源和家族传承
- 以直观的方式查看鸽子的家族谱系图谱
- 在养鸽交流中有据可查，增强鸽友间的互动信任

### 1.3 目标用户画像

#### 1.3.1 核心用户画像

| 属性 | 详细描述 |
|------|----------|
| **年龄范围** | 55-80岁，以60-70岁为核心群体 |
| **性别分布** | 男性为主（约80%），女性为辅 |
| **文化水平** | 初中到高中为主，部分大专及以上，少数小学文化 |
| **职业背景** | 退休人员、个体经营者、农民/城郊居民 |
| **养鸽年限** | 3年以上，部分超过20年的资深鸽友 |
| **饲养规模** | 个人饲养10-200只鸽子不等 |
| **设备情况** | 中低端安卓手机（千元机为主），屏幕5.5-6.5英寸 |
| **网络环境** | 家庭WiFi + 4G/5G，部分用户仅在WiFi环境下使用 |
| **视力状况** | 绝大多数有老花眼（需佩戴老花镜），部分有轻度白内障或青光眼 |
| **触屏操作能力** | 基础操作可完成，复杂手势（长按、滑动、双指缩放）不太熟练 |
| **数字产品使用经验** | 会使用微信、抖音、快手等主流APP的基本功能，对复杂设置感到困惑 |
| **使用习惯** | 喜欢大字体、大按钮、纸质化的视觉感受，不喜欢频繁跳转页面 |
| **核心诉求** | "方便管理大量鸽子，记住每只鸽子的位置和血统" |
| **痛点** | 鸽子多了记不住谁是谁的后代、记不住鸽子放在哪个棚、纸质记录容易丢失损坏 |

#### 1.3.2 次要用户画像

| 属性 | 详细描述 |
|------|----------|
| **年龄范围** | 35-55岁 |
| **特点** | 相对年轻，接受度更高，可能是帮父辈使用或在家庭中主导使用 |
| **使用场景** | 协助老年家人管理鸽子信息，或作为自己养鸽的辅助工具 |
| **诉求** | 希望功能完善、数据可靠，能够为家庭养鸽提供数字化支撑 |

### 1.4 核心价值主张

> **一句话价值**：用简单的方式，管好每一只鸽子。

| 价值维度 | 描述 |
|----------|------|
| **位置管理** | 替代纸质记录，数字化的方式记录每只鸽子的棚舍位置，查找方便 |
| **血统追溯** | 建立完整的家族谱系，让鸽子的血缘关系一目了然 |
| **决策支持** | 基于血统和位置信息，辅助配对、参赛、交易等养鸽决策 |
| **传承记录** | 记录养鸽历程中的鸽子家族历史，形成可追溯的养鸽档案 |
| **降低记忆负担** | 让用户不必再依赖记忆力管理大量鸽子信息，减轻认知负担 |

### 1.5 产品原则

为确保产品设计始终围绕老年用户需求，确立以下产品原则：

| 原则 | 说明 | 示例 |
|------|------|------|
| **大字优先** | 所有文字最小字号不低于16sp，重要信息20sp以上 | 鸽舍名称、鸽子编号使用24sp加粗 |
| **高对比度** | 文字与背景对比度不低于4.5:1，关键操作按钮对比度不低于7:1 | 深色文字配浅色背景，按钮使用深色填充 |
| **一步直达** | 核心操作不超过3步完成，减少页面层级 | 首页直接显示鸽舍列表，点击即查看鸽子 |
| **容错设计** | 操作可随时撤销，关键操作有二次确认 | 删除鸽子前有弹窗确认，删除后可撤销 |
| **纸质感UI** | 界面风格模拟纸质记录本的质感，降低陌生感 | 使用米黄色背景、仿手写体标签、表格线设计 |
| **离线优先** | 核心功能无需联网即可使用，数据本地存储 | 无网络时仍可添加鸽子、查看图谱 |

## 2. 功能需求清单

### 2.1 功能模块总览

| 模块 | 功能名称 | 优先级 | 功能描述 | 用户价值 |
|------|----------|--------|----------|----------|
| **鸽舍管理** | 添加鸽舍 | P0 | 创建新的鸽舍/棚舍，设置名称、位置描述、容纳数量 | 建立管理空间的基本单元 |
| **鸽舍管理** | 编辑鸽舍 | P0 | 修改鸽舍的名称、位置、容量等信息 | 信息有误时可更正 |
| **鸽舍管理** | 删除鸽舍 | P0 | 删除鸽舍（需确认该鸽舍下无鸽子） | 清理不再使用的鸽舍 |
| **鸽舍管理** | 鸽舍列表查看 | P0 | 以卡片/列表形式查看所有鸽舍，显示鸽舍名称和当前鸽子数量 | 一目了然知道有多少鸽舍及各鸽舍情况 |
| **鸽舍管理** | 鸽舍内鸽子查看 | P0 | 进入鸽舍查看该鸽舍下的所有鸽子列表 | 快速定位某鸽舍中的鸽子 |
| **鸽子档案** | 添加新鸽子 | P0 | 录入新鸽子的基本信息（环号、昵称、羽色、性别、入棚日期、照片等） | 建立每只鸽子的数字档案 |
| **鸽子档案** | 查看鸽子详情 | P0 | 查看单只鸽子的完整信息，包括基本信息、当前位置、家族关系入口 | 全面了解一只鸽子的情况 |
| **鸽子档案** | 编辑鸽子信息 | P0 | 修改鸽子的基本信息、更换照片等 | 信息变更时可更新 |
| **鸽子档案** | 删除鸽子 | P0 | 删除鸽子档案（软删除，保留关联记录） | 移除不再饲养的鸽子 |
| **鸽子档案** | 鸽子搜索 | P0 | 通过环号、昵称等关键词搜索鸽子 | 快速找到特定鸽子 |
| **鸽子档案** | 鸽子照片拍摄/上传 | P1 | 拍摄或从相册选择鸽子照片作为档案照 | 直观的视觉辨识 |
| **位置管理** | 分配/变更位置 | P0 | 将鸽子分配到某个鸽舍，或变更其所在鸽舍 | 记录鸽子当前所在位置 |
| **位置管理** | 位置变更历史 | P1 | 查看鸽子曾经的存放位置及变更时间 | 追溯鸽子的位置变迁 |
| **家族关系** | 设置父母关系 | P0 | 为鸽子指定父亲和母亲（从已有鸽子中选择） | 建立血统谱系的基础 |
| **家族关系** | 设置配偶关系 | P0 | 为鸽子指定配偶（一雄一雌配对） | 记录配对信息 |
| **家族关系** | 查看后代列表 | P0 | 查看某只鸽子的所有后代（按出生时间排序） | 了解繁殖成果 |
| **家族关系** | 解除家族关系 | P1 | 删除或修改已建立的父母/配偶关系 | 纠正错误录入的关系 |
| **家族图谱** | 树状图谱展示 | P0 | 以树状结构展示鸽子的祖先（上三代）和后代（下三代） | 直观展示血统传承 |
| **家族图谱** | 配偶关系展示 | P0 | 在图谱中显示鸽子的配偶及其共同后代 | 完整呈现家庭结构 |
| **家族图谱** | 图谱缩放与平移 | P1 | 支持手势缩放和平移大型家族图谱 | 查看复杂的大家族关系 |
| **家族图谱** | 点击跳转 | P0 | 点击图谱中的鸽子卡片可跳转到该鸽子详情页 | 在图谱中浏览不同鸽子 |
| **家族图谱** | 高亮显示 | P1 | 选中某只鸽子时高亮其血缘路径 | 清晰识别血统脉络 |
| **数据备份** | 本地数据导出 | P0 | 将所有数据导出为文件（如Excel/JSON格式）保存到手机存储 | 防止数据丢失，可转移 |
| **数据备份** | 数据导入恢复 | P1 | 从之前导出的文件中恢复数据 | 换手机或误删后可恢复 |
| **系统设置** | 字体大小调节 | P0 | 提供小/中/大/特大四档字体（适配不同视力水平） | 满足不同程度老花眼需求 |
| **系统设置** | 高对比度模式 | P0 | 切换黑白高对比度主题，增强可读性 | 适合白内障或视力严重退化用户 |
| **系统设置** | 操作音效/震动 | P2 | 点击按钮时播放确认音或震动反馈 | 操作反馈更明确 |
| **系统设置** | 使用引导教程 | P1 | 首次使用的图文引导教程 | 帮助老年用户快速上手 |
| **辅助功能** | 语音输入备注 | P2 | 通过语音输入鸽子的备注信息 | 减少打字负担 |
| **辅助功能** | 最近操作记录 | P1 | 显示最近查看和编辑过的鸽子列表 | 方便回到最近操作的鸽子 |
| **辅助功能** | 快捷操作（桌面小部件） | P2 | 在桌面添加小部件，快速打开到指定鸽舍 | 一步直达常用鸽舍 |

### 2.2 优先级说明

| 优先级 | 含义 | 计划版本 |
|--------|------|----------|
| **P0** | 核心功能，MVP必须包含，没有则产品不可用 | V1.0 |

## 3. 用户故事

### 用户故事 1：添加新鸽子

> **作为** 一位养了50多只鸽子的68岁退休工人，  
> **我想要** 在手机上简单快速地添加一只新鸽子的信息，  
> **以便** 我能把每只鸽子的环号和基本信息都记下来，不用再拿纸笔记。

**验收标准**：
- 添加页面每个输入框都有清晰的标签说明
- 环号输入框自动调出数字键盘
- 性别选择使用大按钮单选而非下拉菜单
- 添加完成后有 Toast 提示"添加成功"
- 整个过程不超过5步操作

---

### 用户故事 2：记录鸽子位置

> **作为** 一个有3个鸽棚、经常把鸽子换来换去养72岁老大爷，  
> **我想要** 记录每只鸽子当前在哪个棚里养着，  
> **以便** 我想找某只鸽子的时候不用一个个棚去翻着找。

**验收标准**：
- 鸽子详情页清晰显示当前所在鸽舍名称
- 变更位置只需2步：点击"变更位置" → 选择新鸽舍
- 变更后鸽子立即出现在新鸽舍的列表中
- 位置信息使用醒目的标签卡片展示

---

### 用户故事 3：建立血统关系

> **作为** 一位注重血统、养了15年鸽子的65岁老鸽友，  
> **我想要** 给每只小鸽子指定它的父亲和母亲是谁，  
> **以便** 将来配对繁殖或者跟朋友交流时能说清楚这只鸽子的血统来源。

**验收标准**：
- 设置父母时可以从鸽子列表中选择（支持搜索和按鸽舍筛选）
- 只能选择性别正确的鸽子作为父/母（父为雄，母为雌）
- 设置完成后在鸽子详情页显示父母信息
- 系统自动将该鸽子加入父母的"后代"列表中

---

### 用户故事 4：查看家族图谱

> **作为** 一位想了解某只鸽子"往上是谁的后代、往下出了哪些小鸽子"的70岁鸽友，  
> **我想要** 以一张图的形势清楚看到这只鸽子的全家关系，  
> **以便** 我对这只鸽子的血统脉络一目了然，跟别人聊鸽子时也能说得清楚。

**验收标准**：
- 图谱页面默认以目标鸽子为中心展示
- 上方显示祖先（父、母），下方显示后代
- 配偶显示在目标鸽子旁边，共同后代从配偶关系线延伸
- 每只鸽子用卡片展示（显示环号、昵称、照片缩略图）
- 点击任意鸽子卡片可跳转到该鸽子详情
- 图谱页面文字大小跟随系统字体设置

---

### 用户故事 5：搜索鸽子

> **作为** 一位养了近百只鸽子、记性不太好的75岁老爷爷，  
> **我想要** 只要输入鸽子的环号或者我给它起的名字就能找到它，  
> **以便** 我不用在长长的列表里一个一个去翻着找。

**验收标准**：
- 首页顶部有显眼的搜索框（输入框高度不低于56dp）
- 支持输入环号或昵称进行模糊搜索
- 输入时实时显示搜索结果
- 搜索结果按匹配度排序
- 无搜索结果时显示友好提示"没找到这只鸽子，换个关键词试试"

---

### 用户故事 6：调整字体大小

> **作为** 一位有老花眼、平时看手机需要戴老花镜的66岁阿姨，  
> **我想要** 把App里的字调得大大的、清清楚楚的，  
> **以便** 我不戴眼镜也能看清楚鸽子信息和操作按钮。

**验收标准**：
- 设置页面提供字体大小选项：小/中/大/特大
- "特大"模式下主要文字不小于24sp
- 字体大小变更后立即生效，无需重启应用
- 所有页面文字（包括标签、按钮、输入提示）同步放大

---

### 用户故事 7：数据备份

> **作为** 一位担心手机坏了数据会丢的62岁鸽友，  
> **我想要** 把我记的所有鸽子信息都保存成一个文件，能发给我儿子帮我存着，  
> **以便** 万一手机出问题或者换手机，我的鸽子记录也不会丢。

**验收标准**：
- 设置页面有"导出数据"按钮，点击后选择保存位置
- 导出文件包含所有鸽舍、鸽子、家族关系数据
- 导出过程有进度提示
- 导出文件格式为通用格式（如Excel），可在电脑上打开查看
- 导出的文件可通过微信等方式分享发送

---

### 用户故事 8：快速查看最近操作的鸽子

> **作为** 一位每天需要频繁查看和更新多只鸽子信息的69岁鸽友，  
> **我想要** 打开App就能看到最近我看过和改过的几只鸽子，  
> **以便** 我不需要每次都去搜索或者翻列表，能快速回到正在处理的鸽子。

**验收标准**：
- 首页有"最近查看"区域，展示最近10只操作过的鸽子
- 每个条目显示鸽子环号、昵称、照片和最后操作时间
- 点击条目直接进入鸽子详情页
- 最近操作记录按时间倒序排列

---

### 用户故事 9：大按钮确认操作（防误触）

> **作为** 一位手指不太灵活、经常点错屏幕的78岁老先生，  
> **我想要** 删除鸽子或者改重要信息的时候有大大的确认按钮提醒我，  
> **以便** 我不会不小心点错了把数据弄丢。

**验收标准**：
- 删除操作弹出确认对话框，"取消"和"确认"按钮尺寸不小于48x48dp
- 删除操作可在30秒内撤销
- 重要变更操作（如修改父母关系）有二次确认
- 确认对话框文字清晰说明操作后果

---

### 用户故事 10：纸质化视觉体验

> **作为** 一位用了一辈子纸质笔记本记录鸽子信息、刚开始用手机的71岁老鸽友，  
> **我想要** 这个App看起来像我平时用的笔记本一样亲切，  
> **以便** 我不会觉得这是个冷冰冰的电子产品，愿意每天打开来用。

**验收标准**：
- App背景色采用米黄色/浅米色，模拟纸张质感
- 列表分割线模拟笔记本横线
- 鸽舍卡片采用仿便签纸设计风格
- 整体视觉风格温暖、朴实，不使用冷色调科技感设计

## 4. 核心业务流程

### 4.1 流程一：添加新鸽子的完整流程

**流程目标**：用户将一只新购入或新孵化的小鸽子录入系统，建立完整的数字档案。

**触发条件**：用户获得一只新鸽子，需要在系统中记录。

#### 流程图（文字描述）

```
[首页] 
  │
  ├─ 点击底部"添加"按钮
  │
  ▼

```

  │
  点击某只鸽子
  │
  ▼
[鸽子详情页]
  │
  滑动到"家族关系"区域
  │
  ├─ 点击"编辑父亲"或"添加父亲"
  │     │
  │     ▼
  │  [选择父亲页面] → 选择雄鸽 → 返回详情页
  │  - 详情页父亲位置显示已选鸽子信息
  │  - 系统将该鸽子自动加入父亲的后代列表
  │
  ├─ 点击"编辑母亲"或"添加母亲"
  │     │

#### 家族关系规则

| 规则项 | 规则说明 | 错误处理 |
|--------|----------|----------|
| **性别校验** | 父亲必须是雄鸽，母亲必须是雌鸽 | 选择时自动过滤，无法选择性别不符的鸽子 |
| **自循环校验** | 不能将自己设为自己的祖先 | 选择祖先时排除当前鸽子及其后代 |
| **配偶性别** | 配偶必须是异性 | 选择时自动过滤同性鸽子 |
| **多配偶** | 一只鸽子可以有多个历史配偶（不同时间段） | P1功能：支持设置配对时间段 |
| **后代自动关联** | 设置父母后，系统自动将当前鸽子加入父母的后代列表 | 无需手动双向设置 |
| **关系覆盖** | 修改父母时会覆盖原有关系，并提示"将替换原父亲XX" | 二次确认，保留原关系记录 |

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| **应用启动时间** | 冷启动 ≤ 3秒 | 中低端手机上首次启动不超过3秒 |
| **页面切换响应** | ≤ 300ms | 普通页面切换在300ms内完成 |
| **鸽子列表加载** | ≤ 500ms（100只以内） | 鸽舍内100只鸽子的列表滚动流畅无卡顿 |
| **搜索响应时间** | 实时响应（200ms内） | 输入关键词后200ms内显示搜索结果 |
| **图谱渲染时间** | ≤ 2秒（上下各3代） | 家族图谱加载和渲染不超过2秒 |
| **图片加载** | 渐进式加载，缩略图优先 | 鸽子照片先显示缩略图，清晰图逐步加载 |
| **内存占用** | 峰值 ≤ 200MB | 确保中低端手机（3GB内存）也能流畅运行 |
| **安装包大小** | ≤ 50MB | 考虑到老年用户手机存储空间有限 |

### 5.2 安全性需求

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **本地数据加密** | 数据库文件使用SQLCipher加密存储，防止被其他应用读取 | P0 |
| **导出文件密码保护** | 导出的备份文件支持设置密码（可选），保护隐私 | P1 |
| **隐私权限最小化** | 仅申请必要的权限（存储、相机），不收集用户隐私数据 | P0 |
| **数据隔离** | 应用数据存储在私有目录，其他应用无法直接访问 | P0 |
| **无网络传输** | V1.0版本不涉及云端同步，所有数据本地存储，不上传网络 | P0 |

### 5.3 离线使用需求

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **完全离线可用** | 核心功能（增删改查、家族关系、图谱查看）均无需网络 | P0 |
| **数据本地存储** | 使用SQLite数据库本地持久化存储，不依赖云服务 | P0 |
| **无网络状态提示** | 当检测到无网络时，在设置页面提示"数据导出需要本地存储权限"而非"需要网络" | P1 |
| **离线导入导出** | 数据备份和恢复完全通过本地文件系统进行，不经过网络 | P0 |

### 5.4 数据备份与恢复

| 需求 | 说明 | 优先级 |
|------|------|--------|
| **手动导出** | 用户可主动导出所有数据为文件，保存到手机存储或分享 | P0 |

### 5.5 可访问性需求（适老化设计）

#### 5.5.1 视觉设计

| 需求项 | 具体要求 | 优先级 |
|--------|----------|--------|
| **基础字号** | 默认模式正文16sp，大标题24sp | P0 |

| **高对比度模式** | 开启后对比度 ≥ 7:1，黑白配色 | P0 |
| **行间距** | 正文行间距不小于1.5倍字号 | P0 |
| **按钮尺寸** | 可点击元素最小尺寸48x48dp，主要按钮建议56dp高 | P0 |
| **图标+文字** | 所有功能按钮同时显示图标和文字说明 | P0 |
| **颜色不唯一** | 信息传递不只依赖颜色，同时用文字或图标表达 | P0 |
| **焦点高亮** | 当前选中/聚焦的元素有明显的边框高亮 | P1 |

#### 5.5.2 交互设计

| 需求项 | 具体要求 | 优先级 |
|--------|----------|--------|
| **防误触设计** | 可点击元素间距不小于8dp，避免误触相邻元素 | P0 |
| **操作确认** | 删除、修改重要信息前有二次确认对话框 | P0 |
| **操作撤销** | 删除操作支持30秒内撤销 | P1 |
| **震动反馈** | 关键操作（保存成功、删除确认）提供轻微震动反馈 | P2 |
| **长按替代方案** | 所有长按操作都有点击菜单按钮的替代方式 | P0 |
| **手势简化** | 核心操作不使用复杂手势（如双指旋转、边缘滑动） | P0 |
| **返回逻辑** | 每一步操作都有明确的返回路径，系统返回键行为一致 | P0 |
| **状态可见** | 页面加载、保存中、操作成功/失败都有明确的视觉反馈 | P0 |

#### 5.5.3 认知设计

| 需求项 | 具体要求 | 优先级 |
|--------|----------|--------|
| **语言通俗** | 所有文案使用通俗易懂的日常语言，不用专业术语 | P0 |
| **步骤提示** | 多步骤操作显示当前步骤和总步骤数 | P1 |
| **帮助入口** | 每个页面右上角有"？"帮助按钮，点击显示通俗说明 | P1 |

| **操作说明** | 首次使用功能时显示简短的图文操作提示 | P1 |
| **术语解释** | 涉及鸽界术语的地方（如"回血""杂交"）提供解释 | P1 |

### 6.3 数据约束规则

### 8.1 术语表

| 术语 | 解释 | 备注 |
|------|------|------|
| **足环号** | 戴在鸽子脚上的圆环上的编号，是鸽子的唯一身份标识 | 也叫环号、脚环号 |
| **鸽舍/鸽棚** | 饲养鸽子的棚舍建筑，可以有多个 | 用户可能有赛鸽棚、种鸽棚等 |
| **羽色** | 鸽子羽毛的颜色类型 | 常见：雨点、灰、红、白、黑、花 |
| **回血** | 近亲繁殖，保持血统纯正 | 鸽界常用术语 |
| **杂交** | 不同品系鸽子之间的配对 | 鸽界常用术语 |
| **种鸽** | 专门用于繁殖后代的鸽子 | 非赛鸽 |
| **赛鸽** | 参加比赛的鸽子 | 也叫竞翔鸽 |
| **SP** | Android尺寸单位"可缩放像素" | 与dp类似，但会随用户字体设置缩放 |
| **DP** | Android尺寸单位"密度无关像素" | 保证在不同屏幕密度下显示一致 |
